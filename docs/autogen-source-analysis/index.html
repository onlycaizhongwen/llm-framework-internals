参考 LangChain 分析方式：先讲架构，再讲源码主线。AutoGen 的核心是 Core 消息运行时、AgentChat 高层 API、Extensions 生态适配和 Studio 可视化工具。

总体结论

AutoGen 的源码主线不是单个 Agent 循环，而是一个分层的多 Agent 应用框架：Core 提供消息运行时和底层协议，AgentChat 提供 AssistantAgent 与 Team，Extensions 接入模型和工具，Studio 提供可视化原型入口。

分享时可以这样讲：AutoGen 更像“多 Agent 消息运行时”，而不是单纯的聊天机器人封装。

核心设计思想

运行时优先先有消息、topic、runtime、handler，再有聊天 Agent。
分层 APICore 给框架开发者，AgentChat 给应用开发者，Extensions 接生态，Studio 做可视化。
消息驱动send_message 处理点对点，publish_message 处理 topic 广播。
状态和流式输出Agent 是 stateful，run_stream 输出中间事件和最终结果。
可插拔生态模型、工具、MCP、代码执行都通过抽象接口接入。
声明式组件Component + Pydantic config 支持序列化、Studio 和配置化加载。
