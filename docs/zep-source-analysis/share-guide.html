Zep 源码分享讲稿

面向分享的精读版：先讲仓库边界，再讲 Zep Cloud 接入范式，最后和 mem0 / LangGraph 建立对比。

开场定位

今天讲的 getzep/zep 和 mem0 不太一样。mem0 仓库里能看到本地 memory core，而当前 Zep 仓库 README 明确说它不是 Zep 产品或服务端源码，它主要放 examples、framework integrations、MCP server、evaluation harness 和 deprecated legacy CE。

一句话主线

create user/thread
  -> thread.get_user_context 注入 Context Block
  -> 模型推理 / 工具调用
  -> thread.add_messages 保存本轮对话
  -> graph.search / graph.add 作为按需图谱记忆工具
讲解重点：thread.get_user_context 是自动取长期记忆；thread.add_messages 是把对话写回 Zep；graph.search 是模型按需查 facts/entities/episodes；graph.add 是写业务数据、文档、JSON 或非对话内容。

目录怎么讲

目录分享口径
integrations/源码精读主角，多框架接入包。
examples/使用示例，不是核心实现。
mcp/zep-mcp-server把 Zep graph/thread 能力变成 MCP tools。
zep-eval-harness/记忆效果评测工具链。
ontology/默认 ontology 示例。
legacy/deprecated Community Edition，不作为当前主线。

源码精读口径

仓库边界

证据链：README.md:20 说明仓库是 examples/integrations/tools；README.md:31 指向 Graphiti；README.md:43 说明 legacy CE 已 deprecated。

LangGraph

证据链：context.py:48 是 get_zep_context()；context.py:155 是 build_system_message()；persistence.py:197 是 persist_messages()；store.py:86 是 ZepStore(BaseStore)；tools.py:94 是 create_graph_search_tool()。
讲法：LangGraph 包最完整，覆盖 Context Block 注入、消息持久化、BaseStore 兼容和 graph search tool。ZepStore 的亮点是 hybrid-delegate：backing store 管 exact get/put，Zep graph 管 semantic search。

AutoGen / CrewAI / Vercel

AutoGen 是 Memory 接口，CrewAI 是 storage/tool contract，Vercel AI SDK 是 middleware + onFinish。Zep 没有强推一个统一大抽象，而是按每个框架最自然的扩展点接入。

真实例子怎么讲

用户说：“以后源码分析请用中文，先讲仓库边界，再讲主流程。我正在比较 mem0、Zep 和 LangGraph。”
  1. 新一轮开始：thread.get_user_context(thread_id) 取回用户长期偏好和项目背景。
  2. Agent 生成回答：system message 中带有 Context Block。
  3. 本轮结束：thread.add_messages(thread_id, user + assistant) 保存对话。
  4. 模型需要主动查记忆：调用 graph.search(query, scope="edges")。
这个例子能讲清楚 Zep 和 mem0 的差异：mem0 文档里我们看 memory engine 内核；Zep 当前仓库里我们看 cloud memory service 的 framework adapters。

局限性和边界

边界讲法
不是服务端核心不能从这里证明云端图谱算法细节。
Context Block 在云端组装仓库能看到调用点，看不到服务端组装算法。
异步 ingestion不要假设同一 turn 写后立刻能查。
多框架包分散分享时抽共同范式，不要逐目录念。
长期 memory 隐私要考虑 user/thread 隔离、删除、脱敏和审计。

建议演讲结构

5 分钟：边界澄清。当前仓库是什么，不是什么。
8 分钟：最高层架构。integrations、examples、MCP、eval harness、ontology、legacy。
15 分钟：LangGraph 精读。Context Block、persistence、ZepStore、graph search tool。
10 分钟：其他框架接入。AutoGen Memory、CrewAI Storage/Tool、Vercel middleware/onFinish。
8 分钟：真实例子。一轮对话如何取 context、生成回答、保存 turn、下轮召回。
8 分钟：设计思想和边界。Cloud adapter、graceful degradation、async ingestion、PII-safe logging。
5 分钟：对比。和 mem0 / LangGraph 建立选型地图。

收束口

Zep 当前仓库最值得看的不是一个本地 memory class，而是它如何把云端图谱记忆能力包装成各框架可理解的扩展点。读懂 LangGraph、AutoGen、CrewAI、Vercel AI SDK、MCP 这几条接入线，就能读懂 Zep 在 Agent 生态里的工程定位。
