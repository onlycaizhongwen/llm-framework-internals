总体结论

CrewAI 是一个面向生产级多 Agent 自动化的 Python 框架。它的源码主线不是单纯的“Agent 调 LLM”，而是两套互补抽象：Crews 用 Crew + Agent + Task + Process 表达角色化多 Agent 协作，Flows 用 Flow + @start/@listen/@router 表达事件驱动工作流。

分享时可以这样讲：Crew 解决“谁来做任务”，Flow 解决“流程怎么走”。

核心设计思想

业务语义优先顶层就是 Agent、Task、Crew、Flow，贴近自动化业务表达。
双主线架构Crews 偏自治协作，Flows 偏确定性控制。
分层执行Crew 编排 Task，Task 委派 Agent，Agent 调 Executor/LLM/tools。
装饰器 DSLFlow 用 @start/@listen/@router 把方法变成流程节点。
事件总线横切CrewAIEventsBus 支撑 tracing、streaming、telemetry 和 hooks。
适配器生态LLM 适配 provider/LiteLLM，Tool 标准化外部能力。

分享收束

CrewAI 源码可以浓缩成一句话：Crew 把角色化 Agent 和任务组织成协作团队，Flow 把自动化流程组织成事件驱动链路，LLM、tools、memory、events 都是围绕这两条主线服务的横切能力。
