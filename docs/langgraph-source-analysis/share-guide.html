LangGraph 源码分享讲解稿

适合对外分享的讲法：先定位，再架构，再主流程，最后用源码片段证明设计思想。

开场定位

看 LangGraph 源码时，不要先把它理解成一个 Agent 框架，而要理解成一个状态图运行时。它让我们用节点和边描述流程，用 state 描述上下文，用 checkpoint 保存每一步状态。Agent、工具调用、人类介入、恢复执行，都只是这个图运行时上的不同模式。

目录怎么讲

目录分享口径
graph面向用户的建图 API，核心是 StateGraph。
pregel真正执行引擎，按 Pregel/BSP step 模型运行。
channels状态传播和合并规则。
checkpointcheckpoint 抽象，支撑恢复、回放、中断继续。
prebuilt预构建 Agent 和工具节点。
cli / sdk服务化和远程调用工具链。

主流程一句话

StateGraph 声明图
  -> add_node / add_edge / add_conditional_edges
  -> compile()
  -> CompiledStateGraph
  -> Pregel.stream() / invoke()
  -> loop.tick() 按 step 推进
  -> runner.tick() 执行节点
  -> channel 更新 + checkpoint 保存
LangGraph 的源码脉络很像一个小型编译执行系统。StateGraph 是声明阶段，compile 是编译阶段，Pregel 是运行时。

代码证据

1. 状态图范式

libs/langgraph/langgraph/graph/state.py:130

class StateGraph(Generic[StateT, ContextT, InputT, OutputT]):
    """A graph whose nodes communicate by reading and writing to a shared state.

    The signature of each node is `State -> Partial<State>`.
    """
节点之间不是直接互相调用，而是通过共享 state 协作。

2. Builder 和 Runtime 分离

libs/langgraph/langgraph/graph/state.py:1164

def compile(...) -> CompiledStateGraph[StateT, ContextT, InputT, OutputT]:
    """Compiles the `StateGraph` into a `CompiledStateGraph` object.

    The compiled graph implements the `Runnable` interface and can be invoked,
    streamed, batched, and run asynchronously.
    """
compile 是关键分界线：compile 之前是声明图，compile 之后才是可运行对象。

3. Pregel 回合制执行

libs/langgraph/langgraph/pregel/main.py:2979

while loop.tick():
    for _ in runner.tick(...):
        yield from _output(...)
    loop.after_tick()
每一轮先选出要运行的节点，再执行它们，最后统一更新 channel。

4. Checkpoint 是版本化状态快照

libs/checkpoint/langgraph/checkpoint/base/__init__.py:92

class Checkpoint(TypedDict):
    """State snapshot at a given point in time."""

    channel_values: dict[str, Any]
    channel_versions: ChannelVersions
    versions_seen: dict[str, ChannelVersions]
checkpoint 保存 channel 值、版本、节点看过哪些版本，所以能恢复、回放和中断继续。

最后收束

LangGraph 的源码不复杂，关键是抓住分层。graph 是声明图，pregel 是执行图，checkpoint 是保存图状态，prebuilt 是用图搭好的 Agent 模板。理解这条线，再看中断、恢复、流式、人类介入，就都能落回同一个模型：状态图的版本化执行。
